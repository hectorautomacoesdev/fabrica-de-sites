1. Resumo executivo

Saímos de uma pasta vazia para um Agente Scout funcional: ele mapeia negócios de uma cidade no OpenStreetMap, descobre quais não têm site (ou só têm rede social), pontua a oportunidade de cada um e gera um dashboard HTML navegável (mapa, gráficos, tabela filtrável).

Decisões-chave: começar grátis (OpenStreetMap), sem LLM no núcleo (determinístico = testável e barato) e sem CrewAI por ora (no Python 3.14 ele só resolve uma versão antiga — comprovado por teste).

3. Passo a passo do que fiz

  • Pesquisa de referências. Comparei frameworks (CrewAI/LangGraph/AutoGen), fontes de dados (Places vs Overpass vs scraping), auditoria, geração de sites e prospecção. Achei projetos que fazem algo parecido (ex.: b2b-leads-ai) e validei a abordagem.
  • Reconhecimento do ambiente. Confirmei Python 3.14, Node 24, git. Descobri que a pasta do projeto existia vazia → construí dentro dela.
  • Teste de risco do Python 3.14. Instalei a stack leve (httpx, pydantic, jinja2, typer, rich) — todas funcionam. Dry-run do CrewAI mostrou que ele só resolve a versão 0.11.2 → decisão: fora da Fase 1.
  • Teste de conectividade + fonte. Query real à Overpass para Guarujá retornou HTTP 200 com dados → fonte e internet validadas.
  • Scaffolding do projeto. Estrutura src/, pyproject.toml, .gitignore, .env.example, README, venv.
  • Núcleo. config, models (Pydantic), db (SQLite), taxonomia de sectors.
  • Pipeline do Scout. Fonte plugável (Overpass) → classifier (setor + detecção de presença web) → scorer (score explicável) → insighter (KPIs + insights) → reporter (HTML).
  • CLI com fabrica scout run | setores | stats (Typer + Rich).
  • Testes (pytest): 25 casos. Instalação editável. Smoke test de imports.
  • Execução real contra Guarujá → 385 negócios, banco + dashboard gerados e validados.
  • Reflexão e ajuste. Redefini "lead quente" para exigir contato (268 → 34, número acionável). Re-rodei e re-testei.
  • Documentação. Arquitetura, roadmap, decisões, dúvidas e este relatório.

6. Decisões técnicas (resumo)

  • Fonte grátis: OpenStreetMap/Overpass (sem chave). Serper/Places entram depois como fontes plugáveis.
  • Determinístico primeiro: núcleo sem LLM. IA entra onde agrega (agentes 2–5).
  • Sem CrewAI agora: incompatível na prática com Python 3.14. Plano: venv 3.12 separado nas fases 2+.
  • Score explicável: cada ponto tem motivo. "Só rede social" é o lead mais quente.
  • SQLite + HTML estático: zero infra, zero custo.

Detalhes completos em docs/DECISOES.md.
